1. Project Title

Blockchain Transaction Analytics Dashboard (ERC-20 Tokens)

2. Objective

The objective of this project is to analyze blockchain transaction data on Ethereum and visualize key insights such as transaction volume, gas fees, token distribution, and wallet activity using an interactive dashboard.

3. Problem Statement

Blockchain data is highly complex and difficult to interpret. There is a need for a system that can simplify transaction data and provide meaningful insights for better understanding of blockchain activity.

4. Dataset Description

The dataset used in this project is obtained from Etherscan and contains ERC-20 transaction data including:

Wallet addresses
Transaction value
Gas fees
Token types
Timestamps
5. Technologies Used
HTML, CSS, JavaScript
Tableau (for data visualization)
GitHub Pages (for deployment)
CSV dataset from Etherscan
6. System Design

The system processes blockchain transaction data stored in CSV format and visualizes it using interactive charts. The dashboard includes filters for date range, token type, wallet address, and transaction value.

7. Dashboard Explanation

The dashboard includes:

KPI cards showing total transactions, volume, and gas fees
Line chart for transaction trends
Scatter plot for gas vs transaction value
Bar chart for top tokens
Treemap for volume distribution
Wallet leaderboard for activity analysis
8. Key Insights
Stablecoins like USDT and USDC dominate transaction volume
High-value transactions are concentrated among a few wallets
Gas fees vary significantly depending on transaction value
Transaction activity shows periodic spikes
9. Future Scope
Integration with real-time blockchain APIs
AI-based anomaly detection for fraud analysis
Predictive analytics for transaction trends
10. Conclusion

This project demonstrates how blockchain transaction data can be effectively analyzed and visualized using modern tools. It provides valuable insights into token activity and wallet behavior on Ethereum.
